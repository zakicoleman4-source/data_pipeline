Time-sync regression — what the code does, on your data

residual after the linear fit

Inputs

1. Read every line of recording_*.txt.
Each row: video_ns, UTC, UTC.
N anchors: __N__ span: __SPAN_S__ s

Centering (numeric stability)

2. video_ns ≈ 1012, utc_s ≈ 1.7×109. Squaring blows precision. Centre both: 
x = video_ns - xmean
y = utc_s   - ymean
xmean = __XMEAN__ ns ymean = __YMEAN__ s

OLS slope (drift)

3. Closed form: 
slope = Σ(x·y) / Σ(x·x)
sxx  = Σ(x·x)        ; for σ later
slope ≈ 1e-9 s/ns. Departure from 1e-9 = camera encoder drifting vs Android system clock. 
drift_ppm = (slope·1e9 - 1)·1e6
          = __DRIFT_PPM__ ppm
~7 ppm typical; over 50 min that's ~20 ms accumulated — killed by this slope, not by any single anchor.

Robust trim (MAD)

4. Loop up to 3×: compute residual MAD, drop anchors > 5×MAD, refit. Single anchor can be 400 ms off (writer scheduling). rejected: __NREJ__

Residuals

5. Per anchor: r_i = y_i − (slope·x_i).
RMSE = jitter the linear model couldn't absorb (writer cadence). RMSE: __RMSE_MS__ ms |max|: __MAX_MS__ ms

Cubic check (non-linearity sniff)

6. Fit degree-3 around mean. Compare RMSE. 
cubic_gain = rmse_linear - rmse_cubic
           = __CUBIC_MS__ ms
cubic_gain << RMSE ⇒ relationship is linear; OLS is enough.

Per-query σ

7. OLS prediction-of-mean variance: 
σ(x) = σresid·√(1/n + (x−xmean)²/sxx)
At centroid x = xmean: σ @ centroid: __FIT_MS__ ms
Grows quadratically toward span edges — that's the orange bowl in the main plot.

Per-frame UTC

8. For a frame with PTS t_video_s: 
utc_s = ymean + slope·(t_video_s·1e9 - xmean)
Sub-millisecond accuracy as long as you trust the regression, not any single anchor.
